S0 → S1: Fine-Tuning

Can QLoRA fine-tuning improve a small language model for English-to-Japanese translation? We adapted Qwen 2.5-0.5B-Instruct with task-specific training data and measured the impact across translation quality, terminology compliance, and training stability.

How S1 Was Built from S0
QLoRA Supervised Fine-Tuning Pipeline
S0 — Base Model
Qwen 2.5-0.5B-Instruct used as-is with a translation prompt. No task-specific adaptation. The model receives the English source and generates Japanese using only its pre-trained knowledge.
S1 — Fine-Tuned
Same base model + QLoRA adapters (rank-64, alpha 64, dropout 0.05). Trained on EN-JA pairs formatted as chat-style examples. 4-bit quantized base, only adapter weights updated.

Understanding the Benchmarks
BLEU (char)
Measures character-level n-gram overlap with the reference translation. Higher means the output matches the reference more closely at the surface level. Necessary for Japanese since it has no whitespace word boundaries. However, a fluent re-phrasing can score low even if the meaning is correct.
chrF++
Character n-gram F-score with word-level bigrams. More forgiving than BLEU for Japanese because it captures partial character matches and inflectional variation. A useful complement when BLEU is harsh on valid paraphrases.
COMET
A learned neural metric trained on human quality judgements. Evaluates whether the translation preserves the meaning of the source, not just surface similarity. We treat this as the most important quality metric. High COMET + low BLEU = fluent but differently phrased. Low COMET + high BLEU = copying without understanding.
Terminology Accuracy
Checks whether the approved glossary term appears in the output exactly where expected. A translation can sound fluent while using the wrong term for a domain concept. This metric catches that gap — and it’s where fine-tuning consistently failed to improve.

What We Learned

Fine-tuning works, but with limits. S1 consistently improved BLEU and chrF++ over S0, showing the model learned translation-specific patterns from the training data. The best run (Run A) gained +2.17 BLEU on dev, confirming that QLoRA can adapt even a 0.5B model.

Meaning preservation barely moved. COMET, our primary quality metric, improved by only +0.0012 on test. This means the fine-tuned model produces outputs that look more like the references but don’t actually convey meaning more accurately. The model is learning surface patterns, not deeper understanding.

Terminology accuracy declined in every run. This was the most important finding. Fine-tuning on general translation pairs did not teach the model to use approved glossary terms. In fact, it made it slightly worse (-0.0009 to -0.0067). The model learned general translation fluency but not domain-specific constraints.

Small models are fragile at scale. Run B with a lower learning rate (3e-6) developed NaN instability at later checkpoints. The 0.5B model often starts training normally but breaks down if pushed too long. Conservative settings with early stopping are essential.

Key Findings
Small but real gains
Fine-tuning improved BLEU and chrF++ consistently, but COMET gains were marginal (+0.0012 on test).
Terminology gap persists
Fine-tuning did not improve glossary compliance. Accuracy dropped slightly in every run (-0.0009 to -0.0067).
Training instability at 0.5B
Longer runs are prone to late NaN events. Shorter, carefully monitored runs with early stopping are safer.
Agentic approach wins
S3 with Claude Sonnet 4.6 achieved BLEU 68.75 vs S1's 17.64 — a fundamentally different capability level.
